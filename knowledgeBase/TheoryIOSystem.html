IO System

VotanicXR IO System is a very helpful system for cross platform XR development. User can easy to modify the controls of application without rebuilding the project.

Command Handler (The highest level)

You can map any types of conditions and inputs to a command. If your actions fulfil the command, the command handler will receive this command at the current frame. Besides, you can also send command by scripting or interactions and the handler will receive the command. The API example of command is vCast.Cmd.Received(“CommandName”);

Controller (The second level)

When you have set a Buttons and an Axes to the controller, you can get control by the button and axis index you mapped. The API example of command is vCast.Ctrl.ButtonUp(0);

Device Manager (The third level)

The manager will store all devices ordered by the scan order of device classes you have set in config file. The index of devices will be automatically arranged after scanning device. For example, if you set the scan order is VInput, VRPN and then OpenVR, then the index of devices may be 0-1 (2 VInput controllers), 2 (1 VRPN device) and 3-6 (4 OpenVR controllers). The API example of command is vCast.Input.KeyboardUp(Keycode.Space);

Device Handlers (The lowest level)

Each class of handler will handle its own devices. All handlers are included in device manager. The API example of command is vCast.Input.VInput.ButtonUp(0);

Input types

The main input types are Button Down, Press, Up, Hold and Touch, Axis Value, Down, Press, Up and Hold. The advanced input types are Combo and Count which are the combination of main inputs. These two combination inputs should be set as a command from Setting file.

Tracker

Only VRPN and OpenVR have tracker device. Tracker is used for tracing entity position and rotation such as user head and hand.

Vibration

Only some of VInput and OpenVR devices support vibration. This is a device output.
