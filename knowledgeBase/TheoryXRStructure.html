Theory - XR Structure

  1. SDK Manager

    The top-level gameObject on Hierarchy Window. This gameObject should not be moved. We recommend you keep the transform unchanged. Most of the environment setting are set by SDK manager, such as Editor Config, Performance, Camera Reference and Default Mode.

  2. Camera Reference

    This is an optional setting of a project. The components on the camera reference will be copied to all XR cameras.

  3. Attribute

    There are 6 attributes in the XR world. If you enable on the Inspector Window, the mode will default enable on start.

    TravelerAllow user to move and rotate
    SoulAllow user to ignore physics related to SDK
    DebugAllow user to have debug mode control and check debug message
    TrackingAllow tracker tracking of entities position and rotation
    StereoAllow display 3D images, note that HMD always on
    Test3DDisplay left right eye testing image for checking inverting eyes problem

  4. User Level

    There are 6 user levels used to lock controls and custom use by default. The main purpose is to prevent mistakenly pressing the shortcut of controls by different types of user. Most of the default controls are set on Level 3.

    Note that the user level will be reset to Level 0 every frame. If you want the keep user level, you can enable the User Level Lock. The default use of level lock is Keyboard Mode.  

  5. Frame

    This is the real environment center point of XR world. For example, if you are using CAVE, then CAVE is the frame. Besides, you can set the custom frame parent and start point on Inspector Window.

  6. Users

    Inside the frame, there should be at least one user. Each user can have their own navigation by using the vCast Controller. There are more than 1 way to navigate such as movement and teleport. You can also adjust the parameters on the Inspector Window.

  7. Head and Hand

    These 2 entities are the minimum required entities of a user. Head should be the synchronizer of user to track user actual position and rotation on XR world. Hand should be the navigator for user to navigate to the correct direction.

  8. Controller and Sensor

    The vCast devices of entity.

  9. Boundary

    In the XR system, there are 2 types of boundary. The main purpose of boundary is to prevent user stand and walk on wrong place. Each boundary has their own layer to detect collision and ray cast.

    You have at least 2 ways to create Behaviour Manager:

    1. Menu Bar VotanicXR > vCast >
    2. Right Click on Hierarchy Window, select vCast > GameObject >

    Ground: Detect user correct position Y and allow free falling and raising. The default ground prefab has a red mark for user to adjust the height level, and it will remove on play mode. Wall: Detect user correct position X and Z, block user navigation.

    Note that the boundary can be collider or trigger. The effect of them are different. Collider will totally block your movement.

  10. Start Point

    The automatically generated gameObject when the SDK manager is first time put in the scene. This is the default start point of frame, you can just change the transform of this gameObject to adjust the start point position and rotation. The object under start point are the reference for you to check objects actual size, height of start point and the start point image you should view.

    Note that if the parameter Start as Ground of user is set to be true and there is no ground detected, the user will automatically adjust the height level to start point.
