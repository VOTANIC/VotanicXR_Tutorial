Objective

This chapter introduces the Behaviour System provided by VotanicXR. Behaviour System is a tool to replace coding to implement simple logic behaviour. This chapter introduces the use of Behaviour System by using simple examples of scenario.

Behaviour System in VotanicXR

Behaviour System is a system to manage the child Behaviour, the Behaviour is to implement the simple logic with conditions.

Set up Behaviour System Sample Scene

Go to Assets > Votanic > VotanicXR_Tutorial > Tutorial08_BehaviourSystem and open the scene BehaviourSystem provided.

The scene contains gameObject Behaviour gameObjects and gameObject Chained Behaviour gameObjects, are list of gameObjects for the two sections below.

GameObject Behaviour gameObjects contains a vase prefab vase_01 and box prefab box_wood_01.

GameObject Chained Behaviour gameObjects contains a shield gameObject Shield, box prefab box_wood_01 (chained), a arrow gameObject Arrow and a collisioner prefab Collisioner.

Using Behaviour Manager

Behaviour Manager is the manager of Behaviour System.

Behaviour is unit generated by the Behaviour Manager, to control the behaviour of itself. A Behaviour consists of branches, each branch consists of conditions and actions.

Add Behaviour Manager

The Behaviour Manager Prefab is a gameObject with the V Gear_Behaviour Manager (Script) component only.

To add the Behaviour Manager Prefab,

  1. Right-click vGear > GameObject > Behaviour Manager.

Create Behaviour

Behaviour is the gameObject generated by the Behaviour Manager, to manage the behaviour itself.

To add a Behaviour,

  1. Click Add Behaviour of the manager, a new gameobject is added in [Behaviour], and added into Hierarchy as a child of Behaviour Manager.

  2. Select the Behaviour (0) generated, click Add Condition in the V Gear Behaiour (Script) component. Then click Condition 1 of the Branch 1, it will bring you to the Condition Edit Window.

    There are different types of condition with parameter to choose.

  3. Rename and set the Condition Type to Interaction as follow.

    Attach the box_wood_01 prefab into the Condition 1 .

  4. Click Add Action in the Behaviour component, Action window is brought out with different types of action.

  5. Rename and set the Action Type to Enable as follow.

    Disable the vase_01 prefab and attach it into the Action 1.

  6. The result of this example will be as follow, when the box is selected, the vase appears!

Create Complex Chained Behaviour

By using Behaviour Manager, we can perform scenario without coding.

The following example is a simple scenario, when the box is interacted or we wait in front of the box for five seconds, a shield is found and can be rotate.

The flowchart below shows the logic of the scenario in this section.

  1. The gameObject Chained Behaviour gameObjects provides the gameObjects needed in this section, includes three models and one collisioner.

  2. Here will be two behaviour in this scenario, one to open the box with shield, arrow button and its function, another to rotate the shield.

  3. The first behaviour shows the functions in the branch. If one of the Trigger is triggered, the Plan in the branch will be executed.

    For multiple Plans, one of it will be excuted. The sum of their probability will be 100% automatically.

    For no Trigger, the Plan will be directly executed.

    If none of the Trigger is triggered, the Behaviour will go to next branch.

    Below are the detail of the first behaviour, please attach the corresponding gameObject into condition and action fieldss.

    It performs logical AND in the Trigger Wait in front of box for 5 seconds, and logical OR between the Trigger Wait in front of box for 5 seconds and the Trigger Interact with the box.

    To perform logical AND with two conditions in a Trigger, press the + button next to the first condition.

    To perform logical OR between Triggers, press the + button next to the first Trigger.

  4. The second behaviour shows the usage of loop in the Behaviour.

    Input desire value into Loop Times to loop the behaviour several times.

    Check the option Loop Always to loop the behaviour forever when it being triggered.

    Below is the detail of the second behaviour.

    It performs loop feature when the first Trigger is satisfied. When the Trigger Press Arrow Button is triggered, the Plan Rotate the Shield will be executed.
